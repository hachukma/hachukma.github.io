Unicode Reference (PUA Mapping)

Tone Glyph Metrics (U+E024) – Font‑Internal Values

These are the exact OpenType metrics and anchor coordinates used in the Hachukma-Regular.ttf font to position the tone high diacritic above the base letter. These values were tuned to work correctly in the Windows IME and on the web.

Width:0 (zero advance width – combining mark) Left Side Bearing (LSB):0 Right Side Bearing (RSB):−200 to −309 (negative – overhangs to the left to center the tone) Anchor Type:mark (GPOS lookup “mark”) Anchor Name:top Anchor X:61 (horizontal offset – works with the base anchors at X≈500) Anchor Y:719 (vertical position above the baseline) Glyph Origin:Bottom‑center at (0,0); shape extends upward (negative Y) Base Anchor (on letters):top, X≈500, Y≈top of letter (e.g., -600)

Note: The negative RSB compensates for the anchor X not being exactly 0. If you regenerate the font, consider setting anchor X to 0 and RSB to 0 for cleaner standard behavior – but the current values work perfectly in the IME.
