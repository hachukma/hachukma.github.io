Hachukma IME in Action

Preview of the desktop input method software interface.

The IME shows both Hachukma script and Roman transliteration side by side. Supports custom themes, word count, MonkeyType typing test, and tone diacritic via double‑tap 'p'.

IME Features & Smart Transliteration

The Hachukma IME is built for efficiency and accuracy. Here's what you get out of the box:

Themes

Switch between light, dark, and other available color schemes. Reduce eye strain and choose a look that feels comfortable during long typing sessions.

Font Size

Adjust the displayed Hachukma font size in real time, from small to large, for a more comfortable reading and typing experience.

Visual Error Detection

If a word is typed using an incorrect character sequence, the IME automatically highlights the affected letters in red. This provides instant feedback and helps identify mistakes while learning the correct character forms.

Save & Share

Save your typing preferences to a file and share it whenever needed. Anyone using the same software can load the file and instantly get the same typing experience without having to adjust anything manually.

Unicode Reference (PUA Mapping)

PUA Codepoint Status — Hachukma uses U+E000 to U+E024

These glyphs are mapped to the Unicode Private Use Area (PUA). Hachukma specifically uses the codepoints U+E000 through U+E024.

  • Hachukma has NOT submitted a proposal to the Unicode Consortium to reserve these specific codepoints for Kokborok.
  • Hachukma has NOT been submitted to the ConScript Unicode Registry (CSUR) — registration with CSUR is optional.
  • These assignments are purely private — made solely within the Hachukma font and IME for the purpose of enabling digital typing.
Future : A formal proposal will be submitted to the Unicode Consortium to allocate official, universal codepoints for the Kokborok script.

Tone Glyph Metrics (U+E024) – Font‑Internal Values

These are the exact OpenType metrics and anchor coordinates used in the Hachukma-Regular.ttf font to position the tone high diacritic above the base letter. These values were tuned to work correctly in the Windows IME and on the web.

Width:0 (zero advance width – combining mark) Left Side Bearing (LSB):0 Right Side Bearing (RSB):−200 to −309 (negative – overhangs to the left to center the tone) Anchor Type:mark (GPOS lookup “mark”) Anchor Name:top Anchor X:61 (horizontal offset – works with the base anchors at X≈500) Anchor Y:719 (vertical position above the baseline) Glyph Origin:Bottom‑center at (0,0); shape extends upward (negative Y) Base Anchor (on letters):top, X≈500, Y≈top of letter (e.g., -600)

Note: The negative RSB compensates for the anchor X not being exactly 0. If you regenerate the font, consider setting anchor X to 0 and RSB to 0 for cleaner standard behavior – but the current values work perfectly in the IME.
