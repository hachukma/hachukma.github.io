Hachukma — A Modern Script for Kokborok
Designed for Virtual & Physical Keyboards · 2025–2026

Hachukma is a contemporary writing system I created for the Kokborok language. It is designed to work seamlessly on the physical keyboards you use every day—whether you're writing a document, sending an email, or just typing notes. Developed throughout 2025–2026, Hachukma uses the Private Use Area (PUA) of Unicode, so each character has a unique digital identity and can be displayed on any modern device.

A Writing System That Fits Your Keyboard — Hachukma follows the layout of a standard physical computer keyboard. You don't need a special keyboard or a completely new typing layout. The symbols you already know — like ; ' [ ] , . / \ = - ( ) < > | ` " { } ! @ # $ % ^ & * _ + — stay in their usual places. This means you can type in Kokborok without relearning where the keys are.

Built for Practical, Everyday Use — Hachukma is built for practical, everyday use. The script is designed to work on your computer right now, and the keyboard layout is straightforward. The tone system and digraph auto‑replace are designed to be efficient. It's a script made for the Kokborok language, ready to be used today and in the future.

Hachukma is designed not only for literature and everyday writing, but also for the future of computing and programming in its own script. It preserves familiar punctuation keys (, . / ; [ ] { }) in their standard positions, making it practical for coding while supporting efficient digital typing.

Tone Mark & Punctuation: The tone mark in Hachukma is a phonetic marker, not a punctuation mark. It does not replace or take up the slots of future Kokborok punctuation marks. The tone mark is placed on the p key (via double‑tap), leaving all punctuation keys (;, ', ,, ., /, etc.) completely free for their standard uses and for any future Kokborok punctuation that the community decides to adopt.

Why the Tone Mark is on the p Key (Double‑Tap) — In Kokborok, the tone mark is important for pronunciation and meaning. In Hachukma, you add it by double‑tapping the p key. Why p? Because it keeps all the other keys free for their normal jobs, like punctuation and symbols. This "multi‑tap" approach is fast and intuitive.

Software Experiment — A Note‑Pad for Hachukma

A small note‑pad style software is available at hachukma.github.io/hachukma-IME-software.html. This is not a full IME. It's an early experiment that shows how Hachukma characters can be typed using the multi‑tap system and tone marking.

What about the future? Once official Unicode codepoints are assigned to Hachukma, building a full system‑wide IME (Input Method Editor) will be straightforward. Such an IME would handle the multi‑tap system entirely in software — no special hardware needed. You'd be able to type in Hachukma on any standard keyboard, in any application.

And hardware keyboards? In the future, physical keyboards with Hachukma layouts could be manufactured. These keyboards would work without any external software — the multi‑tap logic and character mapping would be built directly into the keyboard's firmware. Just plug it in and start typing.

Important: The full IME and hardware keyboards have not been built yet. I'm talking about future possibilities. The current note‑pad experiment is a step toward understanding how these systems might work and gathering feedback.

Digraph Maps & Auto‑Replace — How Typing Really Works

One of the core design decisions I made for Hachukma is that typing should feel natural. In Kokborok, certain sounds are represented by two letters together — these are called digraphs: th, ph, kh, ch, and ng. Instead of forcing you to memorise a separate key for each digraph, I built an auto‑replace system right into the typing engine.

Here's how it works: when you type two characters that form a valid digraph, the system automatically replaces the pair with the corresponding Hachukma character. For example:

t + h → 
/tʰ/
p + h → 
/pʰ/
k + h → 
/kʰ/
c + h → 
/tʃ/
n + g → 
/ŋ/

The mapping is simple and consistent. When you type a character that could form a digraph with the previous character, the engine checks the digraph map and performs the replacement instantly. This means you can type t then h, and before you even lift your finger, the two characters have already become a single th character in the Hachukma script.

Why this matters: This auto‑replace feature makes typing in Hachukma feel just like typing in any other language. You don't have to think about "special keys" or "alternate layouts" — you just type the sounds as you hear them, and the system handles the rest.

Under the hood: The digraph map is defined in the JavaScript as digraphMap — it's a simple object that maps second characters to first characters. When you type, the engine checks if the current key + the previous key form a valid digraph, and if so, it replaces the pair. This is the actual production code running in this demo.

Smart Typing with Predictions — To make typing faster, Hachukma includes a prediction engine. As you start typing a word, a list of possible words appears above the text area. For example, typing a will show predictions like ama and awang. You can select a prediction by clicking it or using the arrow keys and Enter. I'll be adding more words to the vocabulary in future updates.

The Building Blocks of Hachukma — The script is built on the sounds of Kokborok. It includes six vowels: a, e, i, o, u, and the central vowel ɘ. It also features 20 consonants that cover the full range of Kokborok sounds. Each character is designed to be visually distinct and easy to learn.
